# CLAUDE.md

Project context and development guidelines for AI-assisted work on this codebase.

## What this project is

Browser-based VS Code ([code-server](https://github.com/coder/code-server)) with Claude, Codex, and Gemini CLIs available in the integrated terminal. Secured with HTTPS (self-signed cert), password authentication, and Squid proxy network isolation so only approved AI API domains are reachable from inside the container.

## Architecture

```
Browser --HTTPS:443--> nginx --HTTP:8080--> code-server --HTTP_PROXY:3128--> squid --> Internet
         frontend (internal)                  isolated (internal)              internet
```

Three Docker networks. Only `squid` has internet access. `code-server` reaches external APIs exclusively via `HTTP_PROXY=http://squid:3128`. `nginx` and `code-server` have no default gateway — isolation is kernel-level, not firewall rules.

## File map

```
docker-compose.yml          — all three services, networks, volumes
.env.example                — template; copy to .env before running
setup.sh                    — first-run: generates password, prompts API keys, builds + starts
code-server/Dockerfile      — extends codercom/code-server:latest, installs Node 20 + CLIs
nginx/Dockerfile            — nginx:alpine + openssl
nginx/nginx.conf            — HTTPS proxy with WebSocket support
nginx/entrypoint.sh         — generates self-signed cert on first start, then execs nginx
squid/squid.conf            — domain allowlist, all else denied
```

## Services

| Service       | Image                        | Port (internal) | Internet access |
|---------------|------------------------------|-----------------|-----------------|
| `nginx`       | built from `nginx/`          | 80, 443 (host)  | no              |
| `code-server` | built from `code-server/`    | 8080            | via squid only  |
| `squid`       | `ubuntu/squid:latest`        | 3128            | yes             |

## Docker networks

| Network    | `internal` | Purpose                      |
|------------|------------|------------------------------|
| `frontend` | true       | nginx ↔ code-server          |
| `isolated` | true       | code-server ↔ squid          |
| `internet` | false      | squid → external APIs        |

## Environment variables (`.env`)

| Variable               | Used by        | Notes                                              |
|------------------------|----------------|----------------------------------------------------|
| `CODE_SERVER_PASSWORD` | code-server    | Plain-text fallback; auto-generated by `setup.sh`  |
| `HASHED_PASSWORD`      | code-server    | Argon2 hash; takes precedence over plain-text      |
| `DOMAIN`               | nginx          | CN + SAN in self-signed cert; default `localhost`  |
| `ANTHROPIC_API_KEY`    | code-server env| Leave blank to use `claude auth login` instead     |
| `OPENAI_API_KEY`       | code-server env| Required; Codex has no OAuth login                 |
| `GEMINI_API_KEY`       | code-server env| Leave blank to use `gemini auth login` instead     |
| `CLAUDE_CONFIG_DIR`    | volume mount   | Host path mounted as `/home/coder/.claude:rw`      |
| `GEMINI_CONFIG_DIR`    | volume mount   | Host path mounted as `/home/coder/.gemini:rw`      |
| `CODEX_CONFIG_DIR`     | volume mount   | Host path mounted as `/home/coder/.codex:rw`       |

## AI CLI packages

Installed globally in `code-server/Dockerfile` via `npm install -g`:

| Binary   | npm package                   | Auth options                          |
|----------|-------------------------------|---------------------------------------|
| `claude` | `@anthropic-ai/claude-code`   | `ANTHROPIC_API_KEY` or `claude auth login` |
| `codex`  | `@openai/codex`               | `OPENAI_API_KEY` or `codex login --device-auth` (ChatGPT subscription) |

Node 20 is installed via NodeSource (`setup_20.x`) because `codercom/code-server` bundles its own internal node which is not on the system PATH.

## Squid allowlist

Domains in `squid/squid.conf` under the `allowed` ACL:

```
.anthropic.com          Claude API + OAuth
claude.ai               Claude subscription OAuth login
.openai.com             Codex API
chatgpt.com             Codex "Sign in with ChatGPT" OAuth
.googleapis.com         Gemini API, Google OAuth (oauth2.googleapis.com, etc.)
.google.com             Google OAuth (accounts.google.com)
registry.npmjs.org      npm registry
.npmjs.com              npm CDN
.github.com             git, extensions
.githubusercontent.com  raw file downloads
.sentry.io              CLI error telemetry
.statsigapi.net         Claude Code feature flags
```

**Squid ACL rules:** Never add both `.foo.com` and `api.foo.com` to the same ACL — Squid treats the shorter suffix as covering the longer one and will refuse to start with "ACL name already defined".

After editing `squid/squid.conf`, apply with:
```bash
docker compose restart squid
```

**Squid logging:** `ubuntu/squid` runs as user `proxy` which cannot write to `/dev/stdout`. Logs go to the `squid_logs` volume at `/var/log/squid/`. To view them: `docker compose exec squid tail -f /var/log/squid/access.log`

## nginx critical settings

Do not remove these — they will break code-server:

- `map $http_upgrade $connection_upgrade` block — required for WebSocket proxying (terminal sessions)
- `proxy_buffering off` — required for real-time terminal output; without it terminals freeze
- `proxy_read_timeout 3600s` — required for idle terminal sessions; default 60s causes silent disconnects
- `proxy_http_version 1.1` — required for WebSocket upgrade handshake

## TLS certificate

Self-signed cert is generated by `nginx/entrypoint.sh` on first container start using `openssl req -x509`. It is stored in the `nginx_certs` Docker volume (not baked into the image) so it persists across rebuilds.

The cert includes a SubjectAltName (`-addext "subjectAltName=DNS:${DOMAIN},..."`) which is required by Chrome 58+ — without SAN the browser will always reject it regardless of CN.

To regenerate the cert (e.g. after changing `DOMAIN`):
```bash
docker volume rm webai_nginx_certs
docker compose up -d nginx
```

## Common commands

```bash
./setup.sh              # first-run setup
./setup.sh --rebuild    # rebuild images without cache and restart

docker compose up -d                        # start stack
docker compose down                         # stop (data persists)
docker compose down -v                      # stop and delete all data
docker compose logs -f                      # tail all logs
docker compose logs -f code-server          # tail one service
docker compose build --no-cache             # rebuild images
docker compose restart squid                # reload squid config
```

## Workspace persistence

VS Code settings, extensions, and workspace files are stored in the `code_server_data` Docker volume mounted at `/home/coder`. They survive container restarts and rebuilds. They are deleted with `docker compose down -v`.

## What NOT to change without understanding the impact

- **`internal: true`** on `frontend` and `isolated` networks — removing this gives code-server direct internet access, defeating the isolation
- **`NO_PROXY=squid,nginx,...`** in code-server env — removing this causes proxy loops (code-server tries to reach squid via squid)
- **`cache deny all`** in squid — enabling caching breaks streaming API responses
- **`proxy_buffering off`** in nginx — re-enabling buffering freezes terminal output
